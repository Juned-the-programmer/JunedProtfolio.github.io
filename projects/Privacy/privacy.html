Terms and Conditions Reading Application

In today's life we are doing lot of transactions a day and we are accepting a terms and Conditions without reading them we will trust the person who is explaining us about that.

For this application we have to create a our own dataset. So for that we have collected the dataset manually from the banks and read everything and differentiate which is against the customer.

We have implemented this in React Native so we can use both in android and IOS devices easily. Used NLP(Word2vec) for model training and implementation.

Used Django Rest Framework for creating a API for that project and used in the application.

This is our minor project so we have worked on this project around 4 months and complete it.

Steps for the Projects are :

1. Take the image of the terms and conditions paper which bank will give us to sign.

2. Upload it to the Our Android Application it will extract the text from that image and store in another variable , USing Django REST framework API.

3. Give that extracted text to the NLP model which will identify which terms and conditions are against the customer and alert the customers on that.

Technologies:
  • - Python
  • - Django REST Framework
  • - React-Native
  • - Google Cloud (For hosting API)
  • - NLP (Natural Language Processing)


It will help the lot of people while doing the sign on the Terms and conditions paper apart from the blindly signing that.

Still we are working on that and updating the Model (NLP) to increase the accuracy of the predictions of the terms and conditions which are aginst the customer.
